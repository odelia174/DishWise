# DishWise – AI Kitchen & Nutrition Assistant

## שם הפרויקט
**DishWise – עוזר תזונה, קניות ובישול מותאם אישית**

## תיאור הפרויקט
DishWise היא אפליקציית Web מבוססת בינה מלאכותית שעוזרת למשתמשים לבשל בבית בצורה חכמה ובריאה יותר.

האפליקציה פותרת בעיות יומיומיות:
- המשתמש לא יודע מה להכין מהמצרכים שיש לו בבית.
- המשתמש רוצה להבין תוויות מזון וערכים תזונתיים.
- המשתמש צריך להימנע מאלרגנים או מרכיבים מסוימים.
- המשתמש רוצה לקבל מתכון מסודר ורשימת קניות.

## קהל היעד
- סטודנטים ואנשים עסוקים שרוצים להכין ארוחות מהירות.
- משפחות שרוצות להפחית בזבוז מזון.
- אנשים עם רגישויות כמו לקטוז או גלוטן.
- אנשים שרוצים להבין טוב יותר את המוצרים שהם קונים.

## כיצד נעשה שימוש בבינה מלאכותית
האפליקציה משתמשת ב-OpenAI כדי:
1. לנהל שיחה טבעית עם המשתמש.
2. לנתח תמונות של מקרר, מזווה או תווית מזון.
3. לשלוף מידע ממאגר ידע מותאם אישית באמצעות RAG.
4. להחזיר מתכונים ורשימות קניות בפורמט מובנה.

## יכולות עיקריות

### 1. Conversational AI
המשתמש יכול לשוחח עם עוזר הבישול ולשאול שאלות כמו:
- "יש לי ביצים, עגבניות וגבינה. מה אפשר להכין?"
- "מה אפשר לשים במקום שמנת מתוקה?"
- "תן לי ארוחת ערב ללא גלוטן עד 20 דקות."

### 2. יכולת מולטימודלית – Vision Input
המשתמש יכול להעלות תמונה של:
- פנים המקרר או המזווה.
- תווית רכיבים או טבלת ערכים תזונתיים.

ה-AI מנתח את התמונה ומחזיר:
- מצרכים שזוהו.
- רכיבים או אלרגנים אפשריים.
- הצעות למתכונים או המלצות זהירות.

### 3. RAG – Retrieval-Augmented Generation
בסיס הידע של המערכת נמצא בתיקיית `data/` וכולל:
- מתכונים.
- מדריך אלרגנים.
- מדריך תחליפי מזון.
- הנחיות תזונתיות כלליות.

המסמכים נטענים אל ChromaDB בעזרת OpenAI Embeddings API, ואז המשתמש יכול לשאול שאלות שהתשובות עליהן מבוססות על המידע שנשלף.

### 4. יכולת OpenAI נוספת – Structured Outputs
האפליקציה מחזירה מתכון ורשימת קניות במבנה JSON קבוע הכולל:
- שם מתכון.
- זמן הכנה.
- מספר מנות.
- רכיבים.
- שלבי הכנה.
- אלרגנים אפשריים.
- רשימת קניות.
- הערות תזונתיות.

## ממשקי OpenAI שבהם נעשה שימוש
- **OpenAI Responses API** – לשיחה עם המשתמש וליצירת תשובות.
- **OpenAI Vision Input** – לניתוח תמונות.
- **OpenAI Embeddings API** – ליצירת embeddings עבור מסמכי הידע.
- **Structured Outputs / JSON Schema** – להחזרת מתכונים בפורמט מובנה.

## התקנה
```bash
git clone https://github.com/odelia174/DishWise.git
cd DishWise
python -m venv venv
```

ב-Windows:

```bash
venv\Scripts\activate
```

ב-macOS / Linux:

```bash
source venv/bin/activate
```

התקנת ספריות:

```bash
pip install -r requirements.txt
```

## משתני סביבה
יש ליצור קובץ `.env` לפי הדוגמה בקובץ `.env.example`:

```env
OPENAI_API_KEY=your_api_key_here
OPENAI_MODEL=gpt-4.1-mini
OPENAI_EMBEDDING_MODEL=text-embedding-3-small
CHROMA_PATH=./chroma_db
```

> אין להעלות את קובץ `.env` או מפתחות API ל-GitHub.

## בניית בסיס הידע

לפני הרצת האפליקציה בפעם הראשונה:

```bash
python build_knowledge_base.py
```

הפקודה תקרא את הקבצים שבתיקיית `data/`, תחלק אותם לקטעים, תיצור embeddings ותשמור אותם ב-ChromaDB.

## הרצת האפליקציה

```bash
streamlit run app.py
```

## הוראות הדגמה

### דוגמה לשיחה עם ה-AI
כתבו בצ'אט:

```text
יש לי אורז, טונה, תירס ועגבניות. מה אפשר להכין לארוחת ערב מהירה?
```

### דוגמה להעלאת תמונה
העלו תמונה של מקרר או מזווה ובחרו:
```text
ניתוח מקרר/מזווה
```

האפליקציה תחזיר רשימת מצרכים שזוהו והצעות למתכונים.

### דוגמה לניתוח תווית מזון
העלו תמונה של גב מוצר ובחרו:
```text
ניתוח תווית מזון
```

האפליקציה תנסה לזהות רכיבים, ערכים תזונתיים ואלרגנים אפשריים.

### דוגמה לשאלת RAG
כתבו:

```text
לפי מדריך התחליפים, במה אפשר להחליף ביצה באפייה?
```

או:

```text
איזה מתכון ללא לקטוז מופיע במאגר?
```

### דוגמה ליכולת Structured Output
בחרו "יצירת מתכון מובנה" וכתבו:

```text
צור מתכון טבעוני מהיר עם אורז, עדשים ועגבניות.
```
